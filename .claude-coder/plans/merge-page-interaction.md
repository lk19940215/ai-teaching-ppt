## 四、状态管理

### 4.1 步骤状态

```typescript
type Step = 'upload' | 'merge' | 'confirm'

interface MergeState {
  currentStep: Step

  // Step 1: 上传
  pptA: File | null
  pptB: File | null
  pptAPages: PptPageData[]
  pptBPages: PptPageData[]
  sessionId: string | null

  // Step 2: 合并设置
  mergeMode: 'full' | 'partial' | 'single'
  globalPrompt: string
  selectedPagesA: number[]
  selectedPagesB: number[]
  mergePlan: MergePlan | null

  // Step 3: 完成
  downloadUrl: string | null
  fileName: string | null
}
```

### 4.2 步骤切换规则

```
upload → merge:
  - 条件：pptA 和 pptB 都已上传且解析成功
  - 动作：自动切换

merge → confirm:
  - 条件：用户点击"确认方案"
  - 动作：调用 generate-final API

任意步骤回退:
  - 保留已输入的数据
  - 清除后续步骤的结果
```

### 8.1 合并提示语交互设计

**位置**：Step 2 顶部控制条（TopControlBar 组件）

**组件代码**：`frontend/src/app/merge/page.tsx` 第 246-321 行

```tsx
function TopControlBar({

```

**交互流程**：

1. 用户进入 Step 2 后，顶部控制条始终可见
2. "合并提示语"输入框位于融合方式选择下方，由分隔线分隔
3. 用户可自由输入提示语，支持多行文本
4. 提示语通过 `globalPrompt` 状态传递到 `handleAiMerge` 函数
5. 最终作为 `custom_prompt` 参数发送到后端 API

**API 调用**：

```tsx
// handleAiMerge 函数中
if (globalPrompt) {
  formData.append('custom_prompt', globalPrompt)
}
```

**后端处理**：`backend/app/api/ppt.py`

```python
@router.post("/ai-merge")
async def ai_merge_ppt(
    custom_prompt: str = Form(""),  # 接收提示语
    ...
):
    # 根据融合模式传递提示语
    if merge_mode == "full":
        result = await merge_full(doc_a, doc_b, custom_prompt)
    elif merge_mode == "partial":
        result = await merge_pages(pages_a, pages_b, custom_prompt)
    elif merge_mode == "single":
        result = await process_single_page(slide, action, custom_prompt)
```

### 8.2 提示语作用场景

| 场景 | 提示语作用 |
|------|-----------|
| 整体合并 | 指导 AI 如何融合两份 PPT 的整体结构 |
| 选择页面 | 指导 AI 如何合并选中的页面 |
| 单页处理 | 指导 AI 如何处理单个页面（润色/扩展/改写/提取） |

### 8.3 设计要点

1. **始终可见**：提示语输入框在 Step 2 顶部，不会被滚动隐藏
2. **可选输入**：标注"可选"提示用户可自由选择是否填写
3. **占位符示例**：提供具体示例帮助用户理解输入格式
4. **禁用状态**：AI 融合进行中时禁用输入，防止误操作

### 8.4 页面级提示语交互

**组件**：`PromptEditor`（`frontend/src/components/prompt-editor.tsx`）

**功能**：
1. **合并策略模板** - 5 个预设模板（保留 A 结构、保留 B 结构、取精华合并、顺序拼接、风格统一）
2. **总体合并策略** - 全局提示语输入
3. **页面级提示语** - 每页可设置"保留"和"废弃"两个字段
4. **提示语汇总** - 实时预览所有提示语

**数据结构**：
```typescript
interface StructuredPagePrompt {
  keep?: string    // 保留内容
  discard?: string // 废弃内容
}

// 页面级提示语存储格式
pagePrompts: Record<string, StructuredPagePrompt> = {
  "A-0": { keep: "保留封面设计", discard: "" },
  "B-2": { keep: "保留例题", discard: "删除重复内容" }
}
```

**后端 API 格式**：
```json
{
  "a_pages": {
    "0": { "keep": "保留封面设计", "discard": "" }
  },
  "b_pages": {
    "2": { "keep": "保留例题", "discard": "删除重复内容" }
  }
}
```

**位置**：右侧面板，与左侧 PPT 预览对应
