## 一、页面概述

/merge 页面是一个 PPT 智能合并工具，采用**三步向导式交互**：

```
Step 1: 上传 PPT → Step 2: 合并设置 → Step 3: 完成下载
```

**核心功能**：
- 上传两份 PPT 文件（A 和 B）
- 通过 AI 提示语指导合并策略
- 支持页面级精细控制（保留/废弃指定内容）
- 生成合并后的新 PPT 文件

## 二、页面布局

### 2.1 整体结构

页面分为三个步骤，用户按顺序完成操作：

| 步骤 | 名称 | 主要操作 |
|------|------|----------|
| Step 1 | 上传 PPT | 上传两个 PPT 文件 |
| Step 2 | 合并设置 | 选择融合方式、编辑提示语、查看预览、确认方案 |
| Step 3 | 完成下载 | 下载合并后的 PPT |

## 四、数据流

### 4.1 状态管理

```typescript
// 步骤状态
currentStep: 'upload' | 'merge' | 'confirm'

// 文件状态
pptA: File | null
pptB: File | null
pptAPages: PptPageData[]
pptBPages: PptPageData[]

// 合并设置
mergeMode: 'full' | 'partial' | 'single'
globalPrompt: string                    // 总体合并策略
pagePrompts: Record<string, {           // 页面级提示语
  keep?: string
  discard?: string
}>

// AI 生成结果
mergePlan: MergePlan | null
adjustedPlan: MergePlan | null
```

### 4.2 API 调用流程

```
用户操作
    │
    ▼
handleAiMerge()
    │
    ├── 构建 FormData
    │   ├── file_a, file_b
    │   ├── merge_type
    │   ├── custom_prompt (globalPrompt)
    │   └── page_prompts (JSON 格式)
    │
    ▼
POST /api/v1/ppt/ai-merge (SSE 流式响应)
    │
    ├── progress: 解析 PPT → 调用 LLM → 生成策略
    │
    ▼
返回 MergePlan
    │
    ▼
用户确认 → POST /api/v1/ppt/generate-final
    │
    ▼
返回下载链接
```

### 4.3 页面级提示语格式转换

前端存储格式：
```typescript
pagePrompts = {
  "A-0": { keep: "保留封面", discard: "" },
  "B-2": { keep: "保留例题", discard: "删除重复" }
}
```

转换为后端 API 格式：
```json
{
  "a_pages": { "0": { "keep": "保留封面", "discard": "" } },
  "b_pages": { "2": { "keep": "保留例题", "discard": "删除重复" } }
}
```

## 五、交互流程

### 5.1 正常流程

1. **上传 PPT**
   - 用户拖拽或点击上传两个 PPT 文件
   - 系统解析 PPT 内容，生成页面预览
   - 自动进入 Step 2

2. **合并设置**
   - 选择融合方式（整体合并/选择页面/单页处理）
   - 在右侧编辑提示语（可选）
     - 点击预设模板快速填充总体策略
     - 或手动输入总体合并策略
     - 或为特定页面设置保留/废弃指令
   - 点击"开始 AI 融合"
   - 查看 AI 生成的合并方案
   - 确认方案或重新生成

3. **完成下载**
   - 显示合并结果信息
   - 下载合并后的 PPT 文件

### 5.2 单页处理流程

1. 在左侧预览区点击某个页面缩略图
2. 弹出操作菜单：润色/扩展/改写/提取
3. 选择操作类型
4. 右侧显示 SinglePageProcessor 面板
5. AI 处理后显示结果预览
6. 用户确认应用或重新生成

### 5.3 回退机制

- 点击"返回上一步"按钮可回退到前一步骤
- 回退时保留已输入的数据
- 从 Step 2 回退到 Step 1：保留上传的文件
- 从 Step 3 回退到 Step 2：保留提示语设置

## 六、融合方式说明

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| 整体合并 | 两份 PPT 全部内容融合 | 合并两个完整课件 |
| 选择页面 | 用户选中部分页面融合 | 只需要部分内容 |
| 单页处理 | 对单个页面进行 AI 处理 | 优化特定页面内容 |
