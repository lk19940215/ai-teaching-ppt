## 根因分析结论

**问题出在 claude-coder 源码的 bug**，而非 Playwright MCP 本身。

`claude-coder` 的 `auth.js`（第 35-41 行）生成 `.mcp.json` 时，只传了 `--storage-state` 却**漏掉了 `--isolated`**。根据 Playwright MCP 官方文档：

- `--storage-state` 的描述是：*"path to the storage state file **for isolated sessions**"*
- 没有 `--isolated` 时，Playwright MCP 运行在**持久化 Chrome Profile 模式**，会直接忽略 `--storage-state`，导致 `playwright-auth.json` 中的 localStorage 数据从未被注入到浏览器中

## 已修复的文件

| 文件 | 修复内容 |
|------|---------|
| `.mcp.json` | args 中添加 `--isolated` |
| `claude-coder/src/auth.js` | 源码修复，新生成的 .mcp.json 将包含 `--isolated` |
| `.claude-coder/playwright-testing-guide.md` | 配置模板和凭证管理说明补充 `--isolated` |
| `.claude/CLAUDE.md` | MCP 工具说明注明 `--isolated + --storage-state` 配对使用 |
| `claude-coder/docs/PLAYWRIGHT_CREDENTIALS.md` | 文档同步修正 |
